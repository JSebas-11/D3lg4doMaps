# v3.0.0 Breaking Changes

:::danger Breaking Changes
This version introduces a **global dependency injection refactor** affecting all SDK packages:

- Core
- Places
- Routes
:::

---

## Dependency Injection Method Renames

Injection methods were simplified and standardized across the SDK.

### Changes

- `AddD3lg4doMaps` → `AddDelgadoMaps`
- `AddD3lg4doMapsPlaces` → `AddDelgadoMapsPlaces`
- `AddD3lg4doMapsRoutes` → `AddDelgadoMapsRoutes`

### Example

Before:

```csharp
services.AddD3lg4doMaps(...);
services.AddD3lg4doMapsPlaces();
services.AddD3lg4doMapsRoutes();
```

After:

```csharp
services.AddDelgadoMaps(...);
services.AddDelgadoMapsPlaces();
services.AddDelgadoMapsRoutes();
```

---

## IOptions Pattern for MapsConfiguration

MapsConfiguration registration now follows the standard ASP.NET Core IOptions pattern.

Before:

```csharp
services.AddDelgadoMaps(new MapsConfiguration {
    ApiKey = "YOUR_API_KEY"
});
```

```csharp
After
services.AddDelgadoMaps(opts => {
    opts.ApiKey = "YOUR_API_KEY";
});
```

---

## RequestTimeout Type Change

MapsConfiguration.RequestTimeout changed from int to TimeSpan.

Before:

```csharp
config.RequestTimeout = 30;
```

After:

```csharp
config.RequestTimeout = TimeSpan.FromSeconds(30);
```

---

## Why These Changes?

- **Cleaner and shorter** DI method names
- **Standardized** configuration patterns
- Better ASP.NET Core **integration**
- Improved **readability and developer experience**
- More **flexible timeout** configuration
